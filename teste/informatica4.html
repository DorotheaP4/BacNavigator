2. (6 puncte) Subprogramul `oglindit` este definit alăturat. Ce valoare va afișa apelul `cout << oglindit(1234);`?

int oglindit(int n) {
    int og = 0;
    while (n > 0) {
        og = og * 10 + n % 10;
        n = n / 10;
    }
    return og;
}

SUBIECTUL al III-lea (30 de puncte)

1. (10 puncte) Scrieți definiția completă a unui subprogram C/C++ `crescator(a, b)` care primește prin parametrii `a` și `b` două numere întregi și returnează `1` dacă `a <= b`, și `0` în caz contrar.

Exemplu: Dacă `a=5, b=8`, subprogramul va returna `1`. Dacă `a=10, b=7`, subprogramul va returna `0`.

2. (10 puncte) Se consideră un tablou unidimensional `v` cu `N` elemente ($N \in [1, 100]$) numere întregi. Scrieți un program C/C++ care citește de la tastatură valoarea `N`, apoi elementele tabloului. Programul va elimina din tablou toate elementele impare și va afișa tabloul rezultat pe ecran, elementele fiind separate prin spații. Ordinea elementelor rămase trebuie să se păstreze.

Exemplu: Pentru `N=7` și tabloul `v = {1, 8, 3, 6, 2, 9, 4}`, se va afișa: `8 6 2 4`

3. (10 puncte) Fișierul `interval.in` conține pe prima linie un număr natural `P` ($P \in [1, 10^5]$), iar pe a doua linie `P` numere naturale din intervalul `[0, 10000]`, separate prin spații. Se cere să se determine și să se afișeze pe ecran numărul de perechi de numere consecutive din fișier care se află în intervalul `[100, 200]`.

O soluție eficientă implică citirea numerelor pe rând și menținerea unei variabile care să rețină numărul anterior citit. Se inițializează un contor la 0. Se citește primul număr. Apoi, într-o buclă, se citește fiecare număr următor. Pentru fiecare pereche (număr anterior, număr curent), se verifică dacă ambele numere se află în intervalul `[100, 200]`. Dacă da, se incrementează contorul. Numărul curent devine numărul anterior pentru următoarea iterație. Această abordare necesită doar stocarea a două numere la un moment dat (cel curent și cel anterior) și o singură parcurgere a fișierului, având o complexitate liniară ($O(P)$) din punct de vedere al timpului și o complexitate constantă ($O(1)$) din punct de vedere al memoriei.

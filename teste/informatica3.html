SUBIECTUL I (20 de puncte)

Pentru fiecare dintre itemii de la 1 la 5, selectați răspunsul corect. Fiecare răspuns corect se notează cu 4 puncte.

1. Ce valoare va avea variabila `x` după execuția secvenței de cod C/C++ de mai jos, dacă `a = 15` și `b = 4`?

int a = 15, b = 4;
int x = a / b + a % b;

2. Considerăm subprogramul `numarCifre` definit alăturat. Ce valoare va returna apelul `numarCifre(1024)`?

int numarCifre(int n) {
    if (n == 0) return 0;
    return 1 + numarCifre(n / 10);
}

3. Câte numere naturale din intervalul `[1, 100]` sunt divizibile cu 3 și nu sunt divizibile cu 5?

4. Se consideră un graf orientat cu 4 noduri. Matricea de adiacență a grafului este dată de:

0 1 0 1
0 0 1 0
1 0 0 0
0 0 0 0

Care este numărul de muchii ale grafului?

5. Se consideră algoritmul alăturat. Dacă se citește `n=5`, care este valoarea afișată?

citește n
s <- 0
pentru i de la 1 la n execută
    dacă i % 2 == 0 atunci
        s <- s + i
    altfel
        s <- s - i
    sfarsit_daca
sfarsit_pentru
scrie s

SUBIECTUL al II-lea (40 de puncte)

1. (28 puncte) Se consideră algoritmul alăturat, descris în pseudocod.

citește n (număr natural)
dacă n == 0 atunci
    scrie 0
altfel
    cât timp n % 10 == 0 execută
        n <- n / 10
    sfarsit_cât_timp
    scrie n % 10
sfarsit_daca

2. (6 puncte) Subprogramul `produsCifre` este definit alăturat. Ce valoare va afișa apelul `cout << produsCifre(205);`?

int produsCifre(int n) {
    if (n == 0) return 1;
    return (n % 10) * produsCifre(n / 10);
}

3. (6 puncte) Definiți în C/C++ o structură `Punct` care să rețină coordonatele `x` și `y` (numere reale). Declarați un tablou de 10 puncte și inițializați primul element cu coordonatele `(3.5, 4.2)`.

SUBIECTUL al III-lea (30 de puncte)

1. (10 puncte) Scrieți definiția completă a unui subprogram C/C++ `estePrim(n)` care returnează `1` dacă numărul natural `n` ($n \in [0, 10^9]$) este prim, și `0` în caz contrar.

Exemplu: Dacă `n=7`, subprogramul va returna `1`. Dacă `n=9`, subprogramul va returna `0`.

2. (10 puncte) Se consideră un tablou unidimensional `v` cu `N` elemente ($N \in [1, 100]$) numere întregi. Scrieți un program C/C++ care citește de la tastatură valoarea `N`, apoi elementele tabloului. Programul va determina și afișa pe ecran, în ordine inversă, toate elementele pare din tablou, separate prin spații.

Exemplu: Pentru `N=5` și tabloul `v = {1, 8, 3, 6, 2}`, se va afișa: `2 6 8`

3. (10 puncte) Fișierul `sir.in` conține pe prima linie un număr natural `M` ($M \in [1, 10^5]$), iar pe a doua linie `M` numere naturale din intervalul `[0, 1000]`, separate prin spații. Se cere să se determine și să se afișeze pe ecran, în ordine crescătoare, toate numerele care apar exact o singură dată în fișier.

O soluție eficientă implică utilizarea unui **vector de frecvență** (`int frecventa[1001]`), inițializat cu zero. Se parcurge fișierul și, pentru fiecare număr citit `x`, se incrementează `frecventa[x]`. După citirea tuturor numerelor, se parcurge vectorul de frecvență de la `0` la `1000` (sau valoarea maximă posibilă a numerelor) și se afișează toate numerele `i` pentru care `frecventa[i]` este egal cu `1`. Această abordare are o complexitate liniară ($O(M + MaxVal)$), unde `MaxVal` este valoarea maximă posibilă a numerelor (1000 în acest caz), fiind eficientă atât ca timp, cât și ca memorie.
